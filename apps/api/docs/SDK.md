# TypeScript Client / SDK Status

Engram's JavaScript/TypeScript package story currently has two tracks:

| Package | Status | Notes |
|---------|--------|-------|
| [`@openengram/mcp`](https://www.npmjs.com/package/@openengram/mcp) | **Published** | MCP server package. Installs the `engram-mcp` binary. |
| `@openengram/client` | **Source available, publish-ready; npm publish pending auth** | Lightweight TypeScript client in [`heybeaux/engram-client`](https://github.com/heybeaux/engram-client). |
| `@openengram/sdk` | **Not published** | Reserved for a future higher-level SDK surface. Do not use in copy/paste instructions yet. |

If you are integrating today, use one of:

1. The REST API directly
2. The published MCP package (`@openengram/mcp`)
3. The TypeScript client source repo (`heybeaux/engram-client`) until `@openengram/client` is published

## REST API Quick Start

Run the API locally:

```bash
git clone https://github.com/heybeaux/engram.git
cd engram
cp .env.example .env
docker compose up -d
```

API base URL:

```text
http://localhost:3001
```

Protected API endpoints require an agent API key generated by the setup/register flow. Send it with `X-AM-API-Key`; use `X-AM-User-ID` to scope the memory owner.

```bash
curl -X POST http://localhost:3001/v1/memories \
  -H "Content-Type: application/json" \
  -H "X-AM-API-Key: <api-key-from-register-response>" \
  -H "X-AM-User-ID: user_123" \
  -d '{"raw": "The user prefers dark mode"}'
```

```bash
curl -X POST http://localhost:3001/v1/memories/query \
  -H "Content-Type: application/json" \
  -H "X-AM-API-Key: <api-key-from-register-response>" \
  -H "X-AM-User-ID: user_123" \
  -d '{"query": "user preferences", "limit": 5}'
```

## Published MCP Package

Install the MCP server from npm:

```bash
npm install -g @openengram/mcp
```

This installs:

```bash
engram-mcp
```

Package details:

- npm: [`@openengram/mcp`](https://www.npmjs.com/package/@openengram/mcp)
- Source: [`heybeaux/engram-mcp`](https://github.com/heybeaux/engram-mcp)
- Binary: `engram-mcp`

## TypeScript Client Source

The lightweight TypeScript client source is available here:

```bash
git clone https://github.com/heybeaux/engram-client.git
cd engram-client
pnpm install
pnpm build
```

Current package metadata in that repo declares the package name as `@openengram/client`, but npm publish is still pending registry authentication.

Example usage once the client is available in your project:

```typescript
import { EngramClient } from '@openengram/client';

const engram = new EngramClient({
  baseUrl: 'http://localhost:3001',
  apiKey: process.env.ENGRAM_API_KEY!,
  userId: 'user_123',
});

await engram.remember('User prefers dark mode');

const memories = await engram.recall('UI preferences', {
  limit: 5,
});
```

Client capabilities include:

- `remember(text, options?)` — store a memory
- `recall(query, options?)` — semantic recall
- `get(id)` — fetch a memory
- `update(id, data)` — update a memory
- `forget(id)` — delete a memory
- `rememberMany(items)` — bulk memory creation
- `generateContext(options?)` — build prompt context
- `dreamCycle(options?)` — trigger consolidation
- `dedupScan()` — scan duplicates
- `health()` — API health check
- `stats()` — memory statistics
- `webhooks.*` — webhook management helpers

## Future SDK

A higher-level `@openengram/sdk` package may exist later, but it is not currently published. Do not add copy/paste install commands or production imports for that package until it is released.

When the SDK ships, this page should be updated with the real npm package name, install command, exports, and verified examples.
